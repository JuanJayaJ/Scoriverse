# Contributing to Scoriverse

Thank you for your interest in contributing to **Scoriverse**! We welcome your input and contributions to improve the package. By contributing, you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md).

## How to Contribute

### Reporting Bugs
- **Search Issues:** Before opening a new issue, please search our [issue tracker](https://github.com/yourusername/Scoriverse/issues) to see if the bug has already been reported.
- **Open an Issue:** If you don't find it, please open a new issue. Include:
  - A clear description of the problem.
  - Steps to reproduce the bug.
  - Expected behavior vs. what actually happens.
  - Any relevant error messages or logs.

### Suggesting Enhancements
- **Feature Requests:** Check existing issues first. If your idea isn’t covered, open a new issue detailing the enhancement and its benefits.
- **Discussion:** Feel free to start a discussion in [GitHub Discussions](https://github.com/yourusername/Scoriverse/discussions) if you're unsure about a feature request.

### Pull Requests
- **Fork and Branch:** Fork the repository and create a new branch for your changes.
- **Coding Standards:** Follow the tidyverse style guide and ensure your code is well-documented using roxygen2.
- **Tests:** Run `devtools::test()` locally and add new tests as necessary.
- **Commit Messages:** Write clear, concise commit messages that describe your changes.
- **Submit PR:** Open a pull request with a detailed description of your changes, linking to any relevant issues.

## Code Style
- Follow the [tidyverse style guide](https://style.tidyverse.org/).
- Ensure new code is well-documented with roxygen2 comments.
- Write tests for any new functionality or bug fixes.

## Communication
- Engage in discussions on GitHub Issues or [GitHub Discussions](https://github.com/yourusername/Scoriverse/discussions).
- If you have questions, feel free to ask in the Issues section or reach out via the designated communication channels.

Thank you for helping to make **Scoriverse** a better project!
